# 📚 Biblioteca Triple Peaks — Landing Page

> Este projeto consiste no desenvolvimento de uma **página de destino (landing page)**
> para a _Biblioteca Triple Peaks_, criada como parte do programa de **Desenvolvimento Web
> da TripleTen**.

O projeto foi desenvolvido de forma **incremental**, evoluindo ao longo de dois sprints
do curso, e representa meu **primeiro projeto independente versionado e publicado no**
**GitHub**, já aplicando práticas profissionais de organização, versionamento e
padronização de código.

---

## 🧭 Contexto do Projeto

### Sprint 2 — Layout de página e Semântica (base conceitual)

Na Sprint 2, o foco foi a construção da base da página, com ênfase em:

- HTML5 semântico
- CSS e Flexbox
- Metodologia BEM
- Criação do cabeçalho e da seção Escolhas da Equipe

Essa etapa serviu como **fundação técnica**, mas ainda não envolvia controle de versão ou
publicação em repositório remoto.

### Sprint 3 — Fluxo de Desenvolvimento (projeto final)

Na Sprint 3, o projeto foi **estruturado do zero em um repositório Git**, tornando-se meu
primeiro projeto publicado no GitHub. Ele foi desenvolvido simulando um fluxo real de
trabalho em desenvolvimento web.

---

## 🚀 Funcionalidades Implementadas

- Página web estática completa
- Navegação funcional com **links de âncora**
- Layout estruturado com **Flexbox**
- Uso de **position: relative e absolute** para composições visuais complexas
- Estrutura HTML semântica
- Rodapé desenvolvido do zero

### Seções da página

- Cabeçalho com navegação
- Escolhas da Equipe
- Eventos (com posicionamento avançado de elementos)
- Seja um membro
- Rodapé institucional

---

## 🛠️ Tecnologias e Ferramentas

- **HTML5** (semântica e acessibilidade)
- **CSS3**
  - Flexbox
  - Posicionamento de elementos
- **Metodologia BEM** para nomenclatura de classes
- **Git & GitHub** (versionamento e publicação)
- **Prettier** para padronização de código
- **ESLint** para boas práticas e consistência
- **Normalize.css** para compatibilidade entre navegadores
